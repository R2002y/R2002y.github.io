Markdown的一些语法

发表于2024-07-13|更新于2024-04-02|内容的格式标准
|字数总计:1.7k|阅读时长:6分钟|阅读量:

之前在写博客的时候，总是忘了它的某些语法是啥，找来找去很麻烦，干脆自己总结一篇，免得再找

一、常见基础语法

一、标题

通过 # 号表示，# 数量对应标题级别（1-6 级），# 后需加空格。

二、文本格式

加粗：用 ** 或 __ 包裹文本

斜体：用 * 或 _ 包裹文本

加粗 + 斜体：用 *** 包裹文本

删除线：用 ~~ 包裹文本

下划线：用  标签（部分编辑器支持 __，但 HTML 标签更通用）

三、列表

  1. 无序列表
    用 -、+ 或 * 开头，后跟空格。
  2. 有序列表
    用数字 + . 开头，后跟空格（数字无需连续，最终会自动排序）。

四、链接与图片

  1. 链接
    格式：显示文本，标题 hover 时显示。
  2. 图片
    格式：（比链接多一个 !）。

五、引用

用 > 开头，可嵌套多层（多组 >）。

六、代码块

  1. 行内代码
    用 ` 包裹（反引号，通常在键盘左上角 Tab 键上方）。
  2. 多行代码块
    用 ``` 包裹，可指定语言（用于语法高亮）。

八、水平线

用 —、*** 或 ___ 表示（至少 3 个字符）。

九、任务列表

用 - [ ] 表示未完成，- [x] 表示已完成（x 不区分大小写）。

一、自定义对齐方式

部分 Markdown 编辑器（如 Typora、Hexo 主题）支持通过 HTML 标签或特定语法设置文本对齐
